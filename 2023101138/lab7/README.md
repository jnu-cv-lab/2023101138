一、问题 1：什么是特征点？
特征点是图像中具有独特性、可重复性的关键点，是图像中最具辨识度的局部区域，能够唯一表征图像中的特定位置，用于后续的特征匹配、目标定位等任务
1.  特征点主要分布在box.png的文字区域、盒子的四个角点、图案边缘以及纹理密集的区域，这些区域的像素灰度变化明显，能够形成独特的局部特征
2.  文字、角点、纹理丰富区域的像素灰度值变化剧烈，存在明显的灰度梯度差异，这种局部的独特性使得ORB算法能够快速检测到这些区域，并提取出具有可区分性的特征点；同时这些区域的特征具有较强的重复性，便于找到对应匹配点
3.  大面积平坦区域的像素灰度值均匀，没有明显的灰度变化和纹理细节，局部区域的特征不具有独特性和可区分性，ORB算法无法检测到具有辨识度的关键点，因此通常没有明显特征点

二、问题 2：什么是特征描述子？
特征描述子就是把特征点周围的颜色、纹理信息，变成一个向量，用来描述这个特征点长什么样，方便在两张图里找到对应的点
1.  区别很简单：关键点是具体的位置（比如某个坐标）；描述子是描述这个特征“长什么样”，是用来匹配的关键。
2.  只知道位置不够，因为两张图里可能有多个位置不同，但长得很像的点，光看位置分不清是不是同一个特征；而描述子能描述每个特征点的形态，对比描述子的相似性，就能准确找到对应点
3.  ORB描述子固定是32维的，不管图像多大、内容是什么，输出的都是32维的二进制向量
4.  ORB用二进制描述子，主要是因为它占空间小、计算快，用简单的异或运算就能对比相似度，匹配起来效率高

三、问题 3：为什么 ORB 使用 Hamming distance？
1.  ORB描述子是32维的二进制向量，每一位只有0和1，每一位都对应特征点周围的颜色对比结果
2.  Hamming距离就是看两个二进制向量有多少位不一样，不一样的位数越少，说明两个特征点长得越像，匹配度越高
3.  ORB不用欧氏距离，因为欧氏距离适合连续的数值向量，而ORB是二进制向量，欧氏距离算不准相似度；而且Hamming距离计算简单、速度快，符合ORB算法“快且实用”的特点

四、问题 4：ORB 为什么对旋转、平移和一定尺度变化具有鲁棒性？
1.  描述子是在特征点周围的小区域里计算的，图像平移后，特征点周围的纹理、颜色关系没变，描述子也不会变，所以ORB能抵抗平移
2. 先算出每个特征点的主要方向，再把特征点周围的区域转到这个主方向，再提取描述子；不管图像怎么转，特征点的主方向也会跟着转，描述子不变，就能抗旋转
3.  抗尺度变化是因为ORB会做不同大小的图像金字塔，在每一层都检测特征点；图像放大或缩小时，总能在某一层找到对应的特征点，描述子也能对应上
4.  ORB不能完全抵抗透视变换。因为透视变换会让物体变形，超出了ORB能适应的范围，光靠ORB匹配不准，必须结合Homography矩阵和RANSAC算法，才能校正变形、完成匹配定位

五、问题 5：为什么初始匹配中会有错误匹配？
1.  错误匹配一般出现在图像有重复纹理、相似图案、背景干扰，或者特征点长得像、光照变化大的时候；比如这次实验里，box_in_scene.png的背景纹理和box.png的部分纹理很像，就容易匹配错
2.  重复纹理、相似图案会让不同位置的特征点长得很像，描述子相似度高，ORB就会误判成正确匹配；背景干扰会引入无关的特征点，这些点和box.png的特征点本来没关系，但偶尔长得像，就会出现错误匹配
3.  匹配距离越小，不一定是正确匹配。因为匹配距离只看描述子像不像，而如果有相似纹理，错误匹配的描述子也可能很像；只有既长得像、位置也符合几何规律的匹配，才是正确的

六、问题 6：RANSAC 的作用是什么？
1.  RANSAC剔除错误匹配的思路很简单：随机从初始匹配里选几个点，算出一个几何模型，再算所有匹配点到这个模型的误差；误差小于5.0的是内点，大于5.0的是错误匹配；重复选几次，选内点最多的模型，保留内点、剔除外点。
2.  RANSAC靠的是“几何一致性”，不是描述子像不像。它不管两个特征点长得多像，只看它们的位置是不是符合Homography矩阵的规律，符合就是正确匹配，不符合就是错误的
3.  内点：符合最优几何模型的匹配点，误差小，是正确匹配
4.  外点：不符合最优几何模型的匹配点，误差大，是错误匹配，会被RANSAC删掉
5.  RANSAC后的匹配线更合理，因为它删掉了不符合几何规律的错误匹配，剩下的内点都能对应上两张图里的同一特征，匹配线不会杂乱交叉

七、问题 7：Homography 的意义是什么？
1.  Homography是一个3×3的矩阵，描述了两张图里同一平面上的点的对应关系，能把一张图里的点映射到另一张图里的对应点，比如处理平移、旋转、缩放和透视变形。
2.  box.png和box_in_scene.png适合用Homography，因为box差不多是平面的，两张图里的box都是同一个平面从不同角度拍的，符合Homography的使用条件
3.  Homography更适合平面物体。因为它描述的是同一平面上点的映射关系，平面物体的所有特征点都在一个平面上，能用一个Homography矩阵对应；立体物体的特征点在不同平面上，一个矩阵对应不了所有点；如果物体不是平面，用Homography会定位不准、匹配出错。因为立体物体不同平面的映射规律不一样，一个Homography矩阵算不准所有点的位置，误差会变大，内点变少，定位就会失败

                                ！！实验总结！！
这次实验我们完成了ORB特征检测与匹配、用RANSAC剔除错误匹配、估计Homography矩阵并实现目标定位，还测试了不同nfeatures参数对实验结果的影响。实验流程很清晰：先用水ORB算法找到box.png和box_in_scene.png的特征点和描述子，用暴力匹配器做初始匹配，再用RANSAC删掉错误匹配，算出Homography矩阵，最后用这个矩阵把box.png的四个角投影到场景图里，框出目标位置。
RANSAC的作用就是删掉初始匹配里的错误点，留下正确的内点，让Homography矩阵算得更准；Homography矩阵是定位的关键，能把模板图的角点准确映射到场景图里，画出目标边框。
实验中我们遇到的主要问题是定位时边框画到了画面外面，后来发现是Homography矩阵的点坐标顺序弄反了，调整顺序后就解决了；另外，初始匹配有很多错误点，用RANSAC处理后，匹配和定位都准确多了。
通过这次实验，我明白了特征匹配的准确性不仅靠ORB这样的检测算法，还需要RANSAC和Homography的配合；特征点不是越多越好，太多会引入噪声，降低匹配准确率，适量就够了
