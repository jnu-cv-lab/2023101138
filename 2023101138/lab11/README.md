1. Transformer 为什么需要位置编码
Transformer 依靠自注意力机制计算特征，这套计算方式本身不会识别输入序列的先后顺序，打乱词语顺序后运算结果不会改变。但文本、时序数据都存在严格语序逻辑，词语位置变化会直接改变整体语义。位置编码就是把每个词语在序列里的先后序号信息添加到词向量中，让模型能够识别序列顺序，正常理解语序带来的语义差异。

2. 传统 sinusoidal position encoding 是如何生成的
按照固定规则分开生成偶数维度、奇数维度的编码数值，偶数维度全部套用正弦计算，奇数维度全部套用余弦计算。不同向量维度使用不一样的振荡频率，高频分量用来区分相邻很近的位置，低频分量用来区分间隔很远的位置，最终生成和词向量维度完全一致、独立存在的位置编码向量。

3. E + pos 的位置注入方式为什么有“内容和位置混合”的问题
词向量E承载语义内容信息，位置编码pos是单独生成的位置信息向量，二者只是简单逐数值相加。两种来自不同特征空间的信息没有做结构化绑定，叠加之后无法再清晰拆分出语义分量和位置分量，互相造成干扰。同时相加只是浅层线性叠加，模型很难自主学到内容和位置的合理关联，而且这套方式绑定固定绝对序号，超长新序列无法适配，内容与位置的融合逻辑直接失效。

4. RoPE 不是加法，而是旋转
传统方案会单独构造位置向量，再和词向量做相加融合。RoPE不会额外生成独立的位置编码向量，直接把原始词向量每两个维度划分为一组，依据当前词语的序列位置算出对应的旋转角度，对每组二维向量执行坐标旋转变换。位置信息直接通过几何变换嵌入进原始语义向量内部，全程不存在两个向量相加的操作。

5. RoPE 作用在 Q 和 K 上
自注意力权重由查询向量Q和键向量K做内积计算得到，数值向量V只负责后续加权汇总特征，不会参与相似度打分。只给Q、K施加旋转位置编码，就能在内积运算里引入位置约束，完整实现相对位置建模；给V做旋转不会改变注意力权重，只会多余增加计算量，因此只在Q、K上执行RoPE。

6. RoPE 的点积天然包含相对位置
分别经过旋转后的Q向量、K向量做内积运算时，各自携带的绝对位置角度会互相抵消，最终运算结果只由两个词语之间相隔的距离决定，和两个词语各自处在序列第几位没有关系。只要两个词语间隔距离相同，无论它们出现在序列哪个区间，内积计算结果基本一致，内积结果里自带相对位置信息。

7. attention score 里的相对位置关系可以通过旋转结构自然出现
注意力打分依靠旋转后的Q、K内积完成，RoPE把位置信息转化成向量的旋转角度。两个词语相隔越近，角度差值就越小，向量方向越接近，内积结果更大，对应注意力权重更高；相隔越远，角度差值越大，内积结果更小，注意力权重更低。不需要额外增加偏置、嵌入等辅助项，仅凭旋转的几何运算结构，注意力分数就自动具备了对词语相对距离的依赖关系。
