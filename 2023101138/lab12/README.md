一、实验概述
本实验基于人体骨架关键点与时序Transformer模型，完成羽毛球六类典型击球动作的自动识别任务，具体包括正手抽球、正手挑球、正手网前球、正手高远球、反手抽球、反手网前球。整体实验流程依次为视频骨架关键点提取、骨架数据预处理、数据集构建、Transformer模型训练与测试评估，最终实现端到端的羽毛球动作时序分类，并结合训练曲线与混淆矩阵全面分析模型性能与存在的问题。

二、方法说明
2.1 数据预处理流程
本实验采用MediaPipe姿态轻量化模型对输入视频进行人体关键点提取，能够稳定输出人体三十三个骨骼关键点信息。每个关键点包含三维坐标与可见度共四维特征，单帧最终可得到一百三十二维骨架特征。为统一不同时长视频的输入规格，实验将所有视频骨架序列统一重采样至三十帧，保证模型输入时序长度一致。
为消除拍摄距离、画面偏移、人体尺度差异带来的干扰，本实验对骨架数据进行归一化处理。以人体左右髋部中点作为坐标原点完成位置平移，消除画面整体偏移误差；以左右肩部距离作为尺度缩放系数，统一不同人物、不同拍摄远近的骨架尺度，让模型仅学习动作姿态差异，不受外部环境干扰。
数据集采用分层抽样的方式按照八比二比例划分训练集与测试集，分层划分可以保证训练集和测试集中各类动作样本分布基本一致，避免类别分布不均导致的评估偏差。为适配少量样本场景，程序设置容错机制，当样本总数过少时不强制划分数据集，全部样本直接用于模型训练。
2.2 模型输入输出规范
模型的输入为预处理完成的标准化骨架时序数据，输入维度包含批次数量、时序帧数与单帧特征维度，统一为三十帧、一百三十二维特征序列。模型输出为六维类别概率向量，分别对应六类羽毛球击球动作，通过概率最大值完成动作类别判定。
2.3 整体实验流程
本次实验整体流程连贯完整，首先读取原始羽毛球动作视频，通过MediaPipe模型提取人体骨架关键点，随后完成骨架归一化、时序重采样的数据预处理操作，构建标准化训练与测试数据集，最终送入Transformer模型完成训练、验证与测试推理，实现羽毛球动作的自动识别与分类。
三、模型结构
本实验采用轻量化的时序Transformer编码器模型，专门适配人体骨架时序特征学习，整体结构简洁规整，主要由特征嵌入模块、位置编码模块、Transformer时序编码模块以及分类输出模块四个核心部分组成，能够高效处理固定长度的骨架时序序列。
特征嵌入模块负责维度转换，将原始一百三十二维的单帧骨架特征映射为一百二十八维高维隐特征，提升模型对动作特征的表征能力，统一模型输入维度规格。由于Transformer自注意力机制不具备时序感知能力，无法识别动作先后顺序，因此模型引入经典正弦余弦位置编码，为每一时序帧添加专属位置信息，让模型能够感知动作的时间变化规律，同时引入少量丢弃概率抑制模型过拟合。
时序编码核心由两层标准Transformer编码器堆叠构成，配备四头多头自注意力机制，能够并行捕捉不同视频帧之间骨架关键点的长距离依赖关系，精准学习羽毛球完整击球动作的时序运动规律。模型内部设置合理的前馈网络维度，通过非线性变换强化特征表达，同时搭配残差连接与层归一化机制，有效避免梯度消失问题，保障模型训练过程稳定收敛。
时序特征编码完成后，模型采用全局均值池化方式，将三十帧的时序特征压缩为单一全局动作特征，融合整段视频的完整动作信息。最后通过多层全连接层组成的分类头完成特征映射，结合激活函数与正则化约束，输出六类羽毛球击球动作的预测概率，完成最终分类任务。
四、训练过程与训练曲线分析
4.1 训练参数配置
本次模型训练统一采用固定超参数配置，训练批次大小设置为十六，初始学习率设置为零点零零一，整体训练轮次共计二十轮。训练过程使用交叉熵损失函数计算分类误差，采用Adam优化器完成网络参数迭代更新，自适应调整学习梯度，保证训练稳定。训练过程中同步记录训练集与验证集的损失值和准确率，自动保存训练过程中性能最优的模型权重用于后续测试评估。
4.2 训练曲线整体分析
本次实验生成的训练曲线包含损失变化曲线与准确率变化曲线，能够直观反映模型的拟合状态与泛化能力。从损失曲线可以看出，训练集损失值随着训练轮次增加持续快速下降，后期逐渐趋于平稳，说明模型能够有效学习训练集中的骨架动作特征，对已知样本拟合效果良好。而验证集损失整体呈现震荡波动趋势，训练后期出现明显上升，与训练损失的下降趋势完全背离，体现出典型的模型过拟合现象。
从准确率曲线可以清晰看出，训练集准确率随训练轮次持续稳步提升，最终达到较高精度，模型充分拟合训练数据。但验证集准确率全程处于较低水平且波动剧烈，无稳定上升趋势，始终远低于训练集准确率，且两者差距随训练推进不断拉大。这一现象充分说明模型在训练样本上拟合效果较好，但无法有效泛化到未知测试样本，泛化能力较弱，过拟合问题较为突出。
整体训练趋势表现为前期模型快速学习动作特征，精度稳步提升，后期模型过度拟合训练集的个性化特征，学习到大量噪声信息，导致在测试数据上的识别性能持续下降。

五、测试结果分析
5.1 整体识别性能
模型在测试集上完成整体评估后，整体识别准确率处于中等水平，能够基本实现六类羽毛球击球动作的分类任务。各类动作的精确率、召回率与F1得分存在明显差异，不同动作的识别效果参差不齐，整体加权平均性能平稳，但相似动作之间的识别误差较大，是影响整体精度的主要原因。
5.2 混淆矩阵结果分析
混淆矩阵可以直观反映各类动作的正确识别数量与错分情况，矩阵对角线代表各类别正确预测样本数量，数值越高代表该类别识别效果越好，非对角线位置代表类别间混淆错误。实验结果表明，动作姿态差异明显、运动特征独特的网前小球类动作识别效果最好，类别混淆极少，模型可以精准区分正反手网前动作。
姿态相似度较高的击球动作容易出现严重混淆，其中正手系列的抽球、挑球、高远球动作骨架轨迹相近，上肢运动姿态重叠度高，模型极易出现错判。同时正反手抽球动作整体运动幅度相似，仅发力肢体存在差异，骨架特征区分度较低，是模型识别误差的主要来源。整体来看，大幅度发力击球动作的混淆概率显著高于小幅网前动作，同类发力方式的动作之间更容易出现识别错误。
5.3 单视频推理效果
训练完成的最优模型可以实现单段羽毛球动作视频的端到端推理识别，对于画面清晰、动作标准、骨架提取完整的视频，模型预测置信度较高，识别结果稳定可靠。但对于动作不完整、画面遮挡、角度偏移严重的视频，容易出现骨架特征缺失的问题，导致预测置信度偏低，出现识别错误。

六、问题分析

在数据层面，本次实验数据集整体样本数量有限，各类动作样本数量分布不均衡，部分类别样本过少，导致模型无法充分学习通用动作特征，容易拟合样本噪声。同时部分原始视频存在拍摄角度单一、光线不均、人体局部遮挡等问题，造成骨架关键点提取不完整、特征存在偏差，进一步降低数据质量，影响模型训练效果。
在模型层面，本次实验存在明显的过拟合问题，模型容量相对现有数据集偏大，Transformer较强的特征拟合能力在小样本场景下容易过度学习训练集个性化特征，丧失泛化能力。同时模型仅使用原始骨架坐标特征，未引入运动速度、关节角度等动态特征，对于静态相似、动态不同的动作区分能力不足，且仅使用全局均值池化融合特征，无法关注动作发力关键帧，时序特征利用不够充分。
在工程应用层面，骨架逐帧提取速度较慢，整体预处理效率偏低。同时模型对非标准动作、残缺动作、多人画面的鲁棒性较差，仅能适配标准单人完整击球视频，实际应用场景适应性较弱。
